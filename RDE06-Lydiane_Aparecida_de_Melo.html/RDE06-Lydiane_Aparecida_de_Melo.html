Software de Versionamento

É o Sistema que tem por objetivo realizar o controle de versões de um código ou documento.
É possível gerenciar as versões de um código ou documento realizadas por um desenvolvedor ou equipe de desenvolvedores.

SOFTWARE DE CONTROLE DE VERSÕES

TIPOS DE SOFTWARE DE CONTROLE DE VERSÃO:

  • Centralizado: Possui um repositório central em um servidor com controle de versões. Ou seja, tudo se conecta ao servidor e fica trabalhando no servidor.
    Dependente do servidor.
    EX:Software Subversion.
  • Distribuído: Possui um repositório local com controle de versões. A pessoa pode ficar trabalhando localmente e quando concluir seu trabalho que vai ocorrer a atualização do repositório Central e necessariamente utilizando um servidor. Atualização no repositório local.
    Independente do servidor.
    EX: Software Mercurial, Git.

GIT

É um Software livre de versionamento que surgiu para o desenvolvimento do Kernel do Linux(Sistema Operacional), foi projeto e desenvolvido por Linus Torvaldes em 2005, o mesmo que criou o Linux.
Cada diretório de trabalho do Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não dependentes de acesso a uma rede ou um servidor central.

Curiosidades: O nome significa "cabeça dura", uma gíria em inglês britânico, que até mesmo o Linus comenta que é um bastardo egocêntrico que nomeou todos os seus projetos com o nome dele.
Mas o que dizem na verdade é que GIT é uma abreviação de um sistema global de rastreamento.

GITHUB

Plataforma de hospedagem de código-fonte e arquivos com controle de versão.
Surgiu em 2008 e foi adquirido em 2018 pela Microsoft por U$ 7,5 bilhões
Hoje hospeda cerca de 100 milhões de projetos com 56 milhões de usuários.

IMPORTANTE:
GitHub é a plataforma de hospedagem.
Git é o programa que vai fazer o controle do versionamento.

Comandos GIT

  • COMMIT
  • PUSH
  • FETCH
  • PULL
